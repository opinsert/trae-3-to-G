# Agent 自动化流程规范

## 1. 代码修改管理流程

### 1.1 提交前检查
- 每次完成代码修改后，必须执行以下检查：
  - 代码通过所有必要的测试
  - 代码通过质量检查（lint/typecheck）
  - 确保没有敏感信息泄露
  - 确保代码符合项目规范

### 1.2 Git 提交规范
- **提交时机**：每次代码修改完成并通过检查后
- **提交信息格式**：
  ```
  [类型] 简短描述
  
  详细说明（可选）
  
  - 修改内容
  - 涉及文件
  - 影响范围
  ```
- **提交类型**：
  - `feat`: 新功能
  - `fix`: 错误修复
  - `refactor`: 代码重构
  - `docs`: 文档更新
  - `style`: 格式调整
  - `test`: 测试相关
  - `chore`: 杂项任务

## 2. 修改记录文档管理

### 2.1 abstract.md 文件
- **位置**：`.trae/documents/abstract.md`
- **生成时机**：项目初始化阶段
- **更新时机**：每次代码修改完成并提交后

### 2.2 abstract.md 文档结构
```markdown
# 项目修改记录

## 修改记录

| 日期 | 修改人 | 修改内容 | Git提交ID | 目的 | 影响范围 |
|------|--------|----------|-----------|------|----------|
| YYYY-MM-DD | 姓名 | 描述 | hash | 说明 | 影响 |
```

### 2.3 更新要求
- 更新内容必须包含：
  - 修改日期（YYYY-MM-DD 格式）
  - 修改人（AI Assistant）
  - 修改内容概述
  - Git 提交记录 ID
  - 修改目的
  - 影响范围

## 3. 执行要求

### 3.1 执行责任人
- **主要执行人**：AI Assistant (Agent)
- **监督人**：项目开发者

### 3.2 执行流程
1. **代码修改阶段**
   - 执行代码修改
   - 进行必要测试
   - 运行代码质量检查

2. **提交阶段**
   - 生成规范的提交信息
   - 执行 git commit
   - 获取提交 ID

3. **文档更新阶段**
   - 打开 `.trae/documents/abstract.md`
   - 添加新的修改记录条目
   - 提交更新（作为补丁提交）

### 3.3 监督机制
- 每次提交后，Agent 应自动检查 abstract.md 是否已更新
- 如发现未更新，应立即进行更新
- 开发者可定期审查 abstract.md 确认修改记录完整性

## 4. 文件位置
- **agent.md**：项目根目录 `/agent.md`
- **abstract.md**：`.trae/documents/abstract.md`

## 5. 注意事项
- 所有 Git 操作需在项目根目录执行
- 提交信息应使用中文或英文，保持一致性
- abstract.md 的更新作为独立提交，提交信息标注 `[docs]`
- 确保敏感信息（密钥、API Keys 等）不会被提交
